A11y purely for your html

Okey so let's get started with some a11y. a11y is a numeronym for accessibility and is massively part of a UX-designers work. However a designers great work could be useless if the code doesn't do what it's supposed to. The content on this page will have CSS to make pretty - but we'll focus on the HTML basic for you developers.

Why accessible?

Okey so the boring answer is: because you're not a terrible person.
Accessibility is useful for most people, and absolutely neccessary for others.
According to that it is bound by discrimation-laws in most countries.

Business point of view

Accessibility is cash! If you run a business you cannot avoid this.
A badly coded website could look as pretty as it wants - but if a user cannot get what they came for it's pointless. We code with a11y in mind - so users with assistive technologies, or invisible disabilities could use the website with ease.
Oh yeah - also it also flies well with google and other search engines!

Users point of view

Every user benefits from this! Simply because the guidelines makes the content better.
Some guidelines to follow are based on mobile-friendly functions, contrasting for screens in sunlight, and not forcing the user to remember or hurry. All in all that makes a website more manageable without timepressure and poorly designed content.

Developers point of view

It makes you aware of what you're coding, and it also makes the next person more likely to understand what you are coding!
What you use for accessibility is as a beginning clean code. Limit your divs, and make the code be what it is supposed to be. It's easy to use the "wrong element" because it demands less fixing in style - but you create code that is hard tpo understand later on.

Examples

Header levels

So first of all - you should use header elements in a programmatical order. Not based on the size they come out in. You could easily fix that with adding the size-styling on a class.

This way you can insert the correct header for it to make sense - and getting the look you want. Bootstrap and other styling libraries made us a bit lazy on this.

Why?

There are many reasons why you should do this. It validates your code properly - and it indicates which text belongs to which headers. If you have a lot of headers nested and the header-levels doesn't make sense, you cannot programmaticly determine which headers are connected and not.

Assistive technologies like screen readers often browse by headers on many websites. By coding them in nested levels they know which to easily skip because they belong together and not.

Guidelines

You should know about and follow WCAG
The newest version is 2.1 and you should follow this to a minimum of AA standard. This is regulated by law in most EU countries - and probably other countries as well.
